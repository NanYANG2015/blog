3.1. Dataset

  • dataset 存储音频路径。

    dataset 缺点
    音频路径 训练时读大量小文件，频繁地寻址，随机读性能慢。
    原始波形或频谱特征 占用另外的存储空间；
    对于输入不同的模型需要分别准备数据；
    训练时 shuffle，变回非连续存储，访问慢

    如 1wh 16kHz、16bit、单声道 采样点数据、80维 FBANK 均约为 1TB。 10000 h * 3600 s/h * 16000 samples/s * 2 byte/sample / (2 ** (10 * 4)) ~= 1.05 TB 10000 h * 3600 s/h * 100 frames/s * 80 dim/frame * 4 byte (float) / (2 ** (10 * 4)) ~= 1.05 TB

  • 另外，WeNet UIO 通过将数据集做成多个压缩包（含音频、文本），shuffle 压缩包，顺序读取一个压缩包内的数据，优化随机读性能慢的问题。

  • dataset 采用 🤗 Datasets，Arrow 格式。

    优点：从磁盘进行内存映射，允许在设备内存相对较小的机器上使用大型数据集。

    存在的问题：🤗 Datasets 基于 Arrow，训练时 shuffle 可达10x 慢。 优化：采用 🤗 IterableDataset，shuffle shards。

  • 特征提取在生成 batch 时计算。

    • 可通过 Dataset.__getitem__ 或 DataLoader.collate_fn 实现。通常，batch 处理更高效，如 🤗 Tokenizers、torch.stft 支持 batch。
    • FBANK 实现：🤗 transformers>=4.40.0 支持 torch.stft GPU batch 计算。
