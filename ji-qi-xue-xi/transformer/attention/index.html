Attention

发布日期:   2025-01-18
更新日期:   2025-02-07

学习序列中不同位置的依赖关系，从而更好地表征语义。

1. attention 算法

1.1. Scaled Dot-Product Attention

\[\operatorname{Attention}(Q, K, V)=\operatorname{softmax}\left(\frac{Q K^T}{\sqrt{d_k}}\right) V\]

  • 提出：Vaswani A. Attention is all you need[J]. Advances in Neural Information Processing Systems, 2017.
  • 出发点：优化高维 \(q\)、\(k\) 下的数值稳定性、训练稳定性。\(q \cdot k=\sum_{i=1}^{d_k} q_i k_i\)，对于较大的 \(d_k\)，点积得到的值可能较大（假设 \(q\)、\(k\) 的元素都是均值为 0、方差为 1 的独立随机变量，则 \(q \cdot k\) 均值为 0，方差为 \(d_k\)），导致 softmax 落入梯度极小的区域，训练不稳定。

1.2. Multi-Head Attention

\[\begin{aligned} \operatorname{MultiHead}(Q, K, V) & =\operatorname{Concat}\left(\operatorname{head}_1, \ldots, \operatorname{head}_{\mathrm{h}}\right) W^O \\ \text { where head } & =\operatorname{Attention}\left(Q W_i^Q, K W_i^K, V W_i^V\right) \end{aligned}\]

  • 提出：Vaswani A. Attention is all you need[J]. Advances in Neural Information Processing Systems, 2017.

  • 出发点：通过多个 attention head，允许模型 attend 不同表示子空间（\(d_k=d_v=d_{\text {model }} / h\)，可以理解为被映射到了子空间）的信息，提高了模型的表征能力。

  • 计算量

    全维度单头 attention \(O\left(d_{\text {model }}^2\right)\)

    multi-head attention \(h \times\left(O\left(d_{\text {model }}^2 / h\right)+O\left(d_{\text {model }}^2 / h^2\right)\right)=O\left(d_{\text {model }}^2\right)+O\left(d_{\text {model }}^2 / h\right)\)。随注意力头的数量增加，计算复杂度会减少。

  • 相较于全维度单头 attention，内存访问效率更高，可以实现更高的计算吞吐量。

    • 多个 head 并行计算，能够充分利用硬件的 SIMD（单指令多数据）能力，减小单次计算的延迟。
    • 单个注意力计算的数据块更小，缓存更友好，尤其是在现代硬件（如 GPU 或 TPU）中，可以更高效地利用寄存器和 L1/L2 缓存。
    • 矩阵切分后的更小子矩阵可以更好地适应硬件的块状存储和预取机制。

2. 计算效率优化

  • 原因：vanilla Transformer self-attention 的二次复杂度。
  • 优化方向
    • 访存高效的注意力算法
      • Multi-Head Attention
      • Flash Attention
    • Linear Attention
      • Lightning Attention-2
    • 部分注意力
      • Local (i.e., sliding window) attention。如 Mistral 7B。
      • alternating local-global attention。如 Character.AI。
      • sparse attention。如 Qwen2.5-Turbo
        • Star Attention
    • KV
      • KV 共享
        • Muti-Query Attention
        • Group-Query Attention (GQA)
        • Cross-Layer Attention
        • Paged Attention
      • KV Cache 压缩
      • ……
    • 多个主机并行计算
      • Ring Attention
      • Star Attention
    • attention head 剪枝
    • ……

2.1. Flash attention

  • 提出：Dao T, Fu D, Ermon S, et al. Flashattention: Fast and memory-efficient exact attention with io-awareness[J]. Advances in Neural Information Processing Systems, 2022, 35: 16344-16359.

    Dao T. Flashattention-2: Faster attention with better parallelism and work partitioning[J]. arXiv preprint arXiv:2307.08691, 2023.

  • https://github.com/Dao-AILab/flash-attention

  • 显著优化了 attention 计算的速度和内存占用，可以以相同的成本训练更长上下文的模型。

  • FlashAttention-3 针对 Hopper GPUs（H100/H800）进行了优化。

2.2. Muti-Query Attention

  • 提出：Shazeer N. Fast transformer decoding: One write-head is all you need[J]. arXiv preprint arXiv:1911.02150, 2019.
  • 所有 Query 共享同一个 Key-Value 对，显著减小了计算量和 KV cache 的存储开销，特别适用于长上下文任务、资源受限设备、实时推理任务。
  • 应用：如 ChatGLM2、Character.AI

2.3. Group-Query Attention

  • 提出：Ainslie J, Lee-Thorp J, de Jong M, et al. Gqa: Training generalized multi-query transformer models from multi-head checkpoints[J]. arXiv preprint arXiv:2305.13245, 2023.
  • 将 query heads 分成 G 个组，每个组内的 query heads 共享一个 Key-Value 对。模型性能和推理速度介于 multi-head attention、muti-query attention 之间。
  • 应用：如 GLM-4、Llama 3.3、Mistral 7B

2.4. Cross-Layer Attention

  • 跨层共享 KV cache。相较于传统的 Multi-Query Attention 没有性能退化，进一步优化了内存占用。

  • 应用：如 Character.AI

    Brandon W, Mishra M, Nrusimha A, et al. Reducing Transformer Key-Value Cache Size with Cross-Layer Attention[J]. arXiv preprint arXiv:2405.12981, 2024.

2.5. Paged attention

  • 提出：Kwon W, Li Z, Zhuang S, et al. Efficient memory management for large language model serving with pagedattention[C]//Proceedings of the 29th Symposium on Operating Systems Principles. 2023: 611-626.
  • https://github.com/vllm-project/vllm
  • 原理

    • 将 KV cache 划分为多个块，每个块包含固定数量 token 的 KV 向量，缓解内存碎片化问题。

      • 避免了为每个请求预先分配一大块连续内存，可以按需动态分配和释放，也避免了新生成 token 时的内存拷贝。
      • 各 KV 块不需要连续存储。

    • vLLM 在前缀一致时，支持请求内（如并行采样（一个请求生成多个答案，如创意写作）、beam search 解码）和跨请求（如具体应用中 prompt 较固定，如机器翻译）KV vache 共享。减少了重复计算与内存占用。

    • 支持更大的 batch size，缩短了请求延迟，增大了吞吐量。

2.6. Ring attention

  • 提出：Liu H, Zaharia M, Abbeel P. Ring attention with blockwise transformers for near-infinite context[J]. arXiv preprint arXiv:2310.01889, 2023.
  • 作者来自 UC Berkeley。
  • 原理
    • 采用 Blockwise Parallel Transformer (BPT)，将输入序列划分为多个块，分别分发给多个主机。每个主机 blockwise 并行计算其输入块的 self-attention 和 FFN，显著减小了内存占用。
    • 环形通信: 所有主机形成环状结构，每个设备在计算 blockwise self-attention 和 FFN 时，将其使用的 KV block 发送给下一个设备，并从上一个设备接收 KV block。因此不需要存储完整序列的 KV 值，进一步减小了内存占用。且通信与计算重叠，没有额外的通信开销。
  • 通信与计算重叠的条件：\(c >= F/B\)。其中，\(c\) 为块大小，\(F\) 为FLOPs，\(B\) 为带宽。推导详见论文第3节。
  • 允许上下文长度随设备数量线性扩展。同时保持高计算利用率，几乎没有额外开销。

2.7. Star attention

  • 提出：Acharya S, Jia F, Ginsburg B. Star attention: Efficient llm inference over long sequences[J]. arXiv preprint arXiv:2411.17116, 2024.

  • 作者来自 NVIDIA。

  • 出发点：许多 LLM 长上下文任务，输入由 long context、short query、short answer 组成。回答 query 通常只需要 context 小部分内容的信息，因此 context tokens 只需关注附近的 tokens，而 query tokens 需要关注之前所有的 tokens。

    对应的 LLM 的 prompt 结构见论文附录 C.3.

  • 两阶段计算

    • Context Encoding：将 context 划分成连续的块，分别分发给多个主机。每个主机计算 anchor block（第一个块）+ 其分配块的 self attention（blockwise-local/block-sparse）。将 attention 计算复杂度从二次降低到线性。相较于 Ring attention，star attention 不与其他主机通信，最小化通信开销。

      anchor block：将第一个 block 作为前缀，使得 blockwise-local attention 的结果接近全局 attention 中存在的 attention sink。实验（论文表 3）表明没有 anchor block 性能显著下降；且相较于采用前一 block，采用第一个 block 性能更好。

    • Query Encoding and Token Generation：各个主机计算 query、其 KV cache 的局部 attention 输出。query host 聚合结果，并更新 query tokens 和新生成 token 的 KV cache。query 和生成 token 能看到全部 context。

      \[ \begin{gathered} A_h=\left(\frac{\exp \left(\frac{Q K_h^{T}}{\sqrt{d}}\right)}{\sum_{k=1}^{l_k} \exp \left(\frac{Q K_{h, k}^{T}}{\sqrt{d}}\right)}\right) V_h \\ s_h=\sum_{k=1}^{l_k} \exp \left(\frac{Q K_{h, k}^{\top}}{\sqrt{d}}\right) \\ A=\left[A_1, A_2, \ldots, A_H\right] \\ s=\left[s_1, s_2, \ldots, s_H\right] \\ s_{\text {global }}=\sum_{h=1}^H s_h \\ A_{\text {global }}=\sum_{h=1}^H \frac{s_h}{s_{\text {global }}} A_h \end{gathered}\]

    • 伪代码详见论文附录 A。

  • block size 设置

    block size 影响准确度、推理速度之间的均衡。建议设置为序列长度的 1/4。取决于具体的用户需求，比如对于超过 128K 的序列长度，block size 固定为 32K 以优先考虑速度；对于更大的模型，如 70B，block size 限制为 16K。

  • 效果

    • 相较于全局 attention，可保持 95-100% 的准确率。在 RULER aggregation 任务（常见词汇提取、频繁词汇提取）上，甚至有显著的性能提升。
    • 相较于 Ring (global) Attention，准确率下降控制在 3% 以内时，可以实现 1.1-11 倍的加速。特别是在长序列、大模型上，速度提升更显著。

  • https://github.com/NVIDIA/Star-Attention

    兼容绝大多数基于全局注意力的 Transformer 模型，无需微调。
