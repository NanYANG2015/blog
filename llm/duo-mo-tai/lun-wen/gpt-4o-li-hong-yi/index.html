LLM 多模态 论文
发布日期:   2024-05-23
更新日期:   2024-07-01

视频地址 https://www.youtube.com/watch?v=CgQ3lUOpXgc&ab_channel=Hung-yiLee

1. audio tokenization

  • audio encoder + codebook，表征语音、人声（喘息声、笑声等）、声音事件等各类声音。
  • 【宏毅老师猜想】混合 token：text token + 非语音 audio token
    • 出发点：若 speech token、text token 独立，有冗余。
    • 【个人想法】若要对齐语音 token 与 text token，需要转写音频数据集，强制对齐音频 token、text token，且会丢失语音段内的非文本信息。
  • speaker diarization

2. audio encoder-decoder

3. 预训练

  • 音频自回归生成 LM
    由于预训练所需音频量较大、音频的录音环境等非常多样，生成音频可能自带 BGM（it is not a bug, it is a feature）。
  • 结合文本数据
    • 由于相较于文本数据，现有的音频数据包含的文本内容（知识）较少，LLM 预训练仍需要结合文本数据。
    • 结合方式：用 LLM 初始化；音频、文本混合训练等。
      • https://arxiv.org/abs/2310.08715
      • https://arxiv.org/abs/2402.05755

4. alignment

  • 训练数据：用户-AI 对话数据

    • 若想生成固定音色，可能不需要大量该音色的训练数据。也可以采用语音转换生成大量指定音色的数据。

  • 语音合成

    • 当有大量的合成训练数据时，合成的声音不再平淡，而是能根据文本内容调整音色。
      BASE TTS (Amazon)：100kh，1B 参数

  • 开始说话、停止说话的判断
    停止说话：听到人声就停止不是最佳方案，比如人与 AI 合唱。

    • 同时输入麦克风输入、历史生成，用户说话时生成 silence，自动预测何时开始说话生成声音 token，用户打断时停止说话。
