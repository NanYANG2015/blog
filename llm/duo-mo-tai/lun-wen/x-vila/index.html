X-VILA

LLM 多模态 论文
发布日期:   2024-06-13
更新日期:   2024-07-11
  • 作者：NVIDIA 等
  • 功能：支持 文本、图像、视频、音频 理解、推理与生成。
  • 创新点
    • 在 vision-to-vision 生成任务中，由于将 visual encoder 表示投影到 LLM 的表示空间, 而 LLM 更倾向于表征语义概念，丢失了视觉细节。提出 Visual Embedding Highway (VEH)，连接 visual encoder 与 decoder，以增强 visual 输入、输出的一致性。
    • any-to-any 模态交错的指令集。
    • 3 阶段对齐训练、指令微调。

1. 系统结构

  • encoder
    • 文本：tokenizer
    • 音频、图像、视频：ImageBind
  • LLM：Vicuna-7B-1.5
  • decoder：扩充 LLM 词表，添加特定模态的 token。提取生成的 token 序列对应的 embedding 序列，通过 transformer 层投影到 decoder 的文本 encoder 特征空间。decoder 为 diffusion 模型，cross attention 文本 encoder 表示。
    • 音频：AudioLDM
    • 图像：Stable Diffusion 1.5
    • 视频：VideoCrafter2

1.1. VEH

  • VEH embedding \(\mathbf{E}^{\mathrm{vis}}\)：visual encoder feature map 逐层相加。

  • RB：2个残差卷积。

  • ZC：全0初始化，生成输出控制信号。

  • visual decoder 在 reverse step \(t\)： \[ \epsilon^p=\left\{\begin{array}{ll}\mathrm{U-Net}_m\left(z(t), \operatorname{VisCtrl}_m\left(\mathbf{E}^{\text {vis}}\right), \mathbf{E}_m^{\text {text}}\right) & \text { if } t<T \times \alpha \\ \mathrm{U-Net}_m\left(z(t), \text {Null}, \mathbf{E}_m^{\text {text}}\right) & \text { if } t \geq T \times \alpha\end{array}, m \in\{\text {'image', 'video' }\}\right. \]

    其中，\(\epsilon^p\) 为预测的 noise，\(z(t)\) 为输入 latent，\(T\) 为 diffusion 步数，NULL 表示不输入 VEH 特征。

  • 参考文献

2. 训练

  • encoder-LLM-decoder 文本对齐训练
    • 数据集：LLaVA-pretrain, cc3m, WebVid, AudioCaps, WavCaps，X-text 对，字幕。
    • 可训练参数：输入投影层、LLM embedding、输出投影层。进行 X-to-text、text-to-X 训练。
    • 训练目标：text-to-X：最小化输出投影层表示、diffusion 模型原始预训练的 text encoder 的表示的距离。
  • 模态交错数据 预训练
    • 数据集：MMC4 图文交错、ActivityNet Captions

      • ActivityNet Captions：视频，自建多模态交错数据，构建方式如下图。

    • 训练：输入投影层、输出投影层、LLM LoRA。
    • 作用：减轻灾难性遗忘、长上下文理解能力。
  • X-to-X 跨模态指令微调

    • 数据集：基于视频字幕数据集构建。每个对话包含至少 1个跨模态问答对。

    • 训练：两阶段

      • 文本对齐

        • 对于图像、视频、音频输出，优化模型生成的 \(\mathbf{E}_m^{\text {text}}\) 与 diffusion 模型 encoder 表示的距离。
        • 可训练参数：输入投影层、输出投影层、LLM embedding、LoRA。

      • 视觉对齐

        • 可训练参数：视觉控制模块、视觉 decoder。

3. 评价

  • 评价指标：X-to-X Alignment Score (\(\mathrm{X}^2 \mathrm{A}\) Score)
    • 采用 ImageBind transformer 分别提取 ground truth、预测的 embedding，计算余弦相似度。
实验 结果 结论
多模态 benchmark 与专门的 VLM 性能接近。
VEH 有效性 相较于 Next-GPT，有显著的性能提升。
添加 VEH，图像、视频生成的一致性增强。
VEH \(\alpha\) 随 \(\alpha\) 增大，生成图像的一致性增强。
LLM 输出与 decoder 对齐
1. 文本对齐：LLM 生成文本描述；
2.Text-Embed-Aligned: LLM 生成特定模态的 token，取对应的 embedding；
3.+ VEH		 结果见上图 文本对齐：纯文本描述很难传达视觉风格、对象外观、精确的人类动作等具体细节。embedding 提供了更大的信息量。
  • 涌现能力
    • 组合不同模态的多个输入，生成一致的内容。
    • 图像-音频、音频-图像 生成。
