LLM 多模态 多模态表示 论文
发布日期:   2024-07-14
更新日期:   2024-07-14
  • 作者：Meta AI
  • 发表时间：2023.05
  • 核心思想
    • 仅采用与图像自然配对的无监督数据（除图文数据外）、对比 loss，训练6种模态：视觉、文本、音频、深度、thermal、Inertial Measurement Unit (IMU) 的联合 embedding 空间。不需要所有模态对齐的数据，实现所有模态的 embedding 对齐。
    • 可用于跨模态检索、embedding 空间的运算、跨模态生成。

1. 数据

模态 数据集
网络图文
视频、音频 Audioset
图像、深度 SUN RGB-D
图像、thermal LLVIP
图像、IMU Ego4D

*由于 SUN RGB-D、LLVIP 数据集较小，复制 50 倍。

*IMU 数据：egocentric cameras video with IMU。

2. 训练

2.1. 对比学习 loss

InfoNCE[54] loss \[L_{\mathcal{I}, \mathcal{M}}=-\log \frac{\exp \left(\mathbf{q}_i^{\top} \mathbf{k}_i / \tau\right)}{\exp \left(\mathbf{q}_i^{\top} \mathbf{k}_i / \tau\right)+\sum_{j \neq i} \exp \left(\mathbf{q}_i^{\top} \mathbf{k}_j / \tau\right)}\]

其中，\(\mathbf{q}_i\), \(\mathbf{k}_i\) 为成对的 image-x 对经过 encoders 得到的 embedding 表示，\(\tau\) 为温度标量，将 mini-batch 内所有 \({j \neq i}\) 的样本视为负样本。实际使用对称 loss \(L_{\mathcal{I}, \mathcal{M}}+L_{\mathcal{M}, \mathcal{I}}\)。采用上述 loss 优化 encoders。

由于图像 encoder 冻结，对比仅采用 L2 loss、L2+对比 loss、仅采用对比 loss，仅采用对比 loss 性能最好。

2.2. encoder

  • 所有模态 encoder 均采用 Transformer 结构。可用预训练模型初始化。
    • 图像：ViT [13]。
    • 视频：与图像采用同一 encoder [20]，temporally 膨胀[7] ViT 的 patch 投影层，采样 2帧/2s。
    • 音频：参照 [22]，mel 谱，ViT-B，patch size=16, stride=10。
    • thermal、深度：视为单通道图像，ViT-S。遵循 [21] 将深度图转为 disparity maps 以实现尺度不变性。
    • IMU：加速度计、陀螺仪在 X、Y、Z 轴上的测量值，2000/5s，1D 卷积 + Transformer。
    • 文本：CLIP [60]。
  • 各模态特定的 encoder（除视频外） + 线性层，以获得相同维度的 embedding。归一化。
  • 采用 OpenCLIP 预训练的视觉、文本 encoders (ViT-H 630M params、302M params)，在 billions 图文对上训练。
  • 视觉、文本 encoder 冻结，音频、thermal、深度、IMU encoder 更新。

2.3. 超参数

  • batch size：每个 minibatch 采样一对模态。通常对比 loss 需要更大的 batch size，但不会随模态数增加而增加。

3.3. 消融实验

encoder 具体配置详见 Sec.5。

  • 缩放图像 encoder 的大小

    • 随着视觉 encoder 性能提升，其它非视觉模态的 emergent zero-shot 分类性能提升。

  • 研究不同的训练设计对深度图（视觉+空间）、音频（非视觉+时序） zero-shot 分类性能的影响，以验证方案的鲁棒性、可迁移性。

    • (a) 对比 loss 温度（参照 [60]）

      • 相较于可学习的温度（初始值 0.07），固定温度更好；
      • 对于深度、thermal、IMU，较大的温度值较好；对于音频，较小的温度值较好。

    • (b) encoder 投影层

      • 相较于 MLP，线性层更好。

    • (c) 训练 epoch

      • 随 epoch 增加性能持续提升。

    • (d) 图像数据增强

      • basic：裁剪、颜色抖动；strong：+ RandAugment、RandErase。
      • 对于数据集较小的深度数据，强增强有助于性能提升；
        对于音频，basic 增强有助于性能提升，强增强性能下降。

    • (e) 对于图像-深度对，相较于使用空间对齐的随机裁剪，使用非对齐的随机裁剪显著降低了性能。

    • (f) 对于图像-深度对，RandomErase 数据增强提升了性能。

    • (g) 对于视频-音频对，使用时间对齐的样本性能更好。

    • (h) 对音频进行频率 mask 数据增强，有少量的性能提升。

    • 音频、深度 encoder 容量大小

      • 大音频 encoder + 大图像 encoder。
      • 小深度 encoder + 大图像 encoder，由于深度数据集较小。

    • batch size

      • 根据数据集大小、复杂度，不同模态的最优 batch size 不同。

  • 采用 IMAGEBIND 评价预训练视觉模型用于多模态任务的性能。

    • 采用不同的预训练视觉 encoder，冻结，使用图像配对数据训练，测试 zero-shot 分类性能。

    • 有监督模型 DeiT 在图像分类任务上更好，自监督模型 DINO 在深度、音频分类任务上更好。
