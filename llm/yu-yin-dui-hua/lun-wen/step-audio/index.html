Step-Audio

LLM 语音对话 论文
发布日期:   2025-02-27
更新日期:   2025-02-27

  • Huang A, Wu B, Wang B, et al. Step-Audio: Unified Understanding and Generation in Intelligent Speech Interaction[J]. arXiv preprint arXiv:2502.11946, 2025.

  • 作者来自阶跃星辰。

  • https://github.com/stepfun-ai/Step-Audio 开源了模型、上线了跃问 APP。

  • 功能：全双工语音交互，支持多语种（中、英、日）、方言（粤语、四川话）、生成音频的细粒度控制、唱歌（干声、RAP）、工具调用。

  • 创新点

    • audio Tokenizer：双码本，离散 token 2:3 交错。作为 LLM 输入，用于音频理解；作为 TTS LM 输出、flow matching 输入，linguistic token 有助于控制音色、韵律，semantic token 有助于生成语义连贯的语音。
      • linguistic tokenizer: Paraformer encoder，提取音素和语言学信息，16.7Hz，码本大小 1024。
      • semantic tokenizer：CosyVoice 的 SenseVoice-Large encoder，提取语义和粗粒度声学信息，25Hz，码本大小 4096。
    • 实时推理：VAD，检测到用户说话，流式提取音频 token；若用户短暂暂停，生成推测性响应；若用户继续说话，丢弃推测性响应；若判断用户说话结束，采用最新的推测性响应，合成音频。若用户打断，处理新输入。ASR 异步转写用户语音，对话历史采用文本替换音频。实验表明，约 40% 的推测性响应被执行，响应延迟减少了约 500ms（VAD 区分短暂暂停、说话结束的延迟差）。待改进：基于 VAD，需要进一步区分 backchannel、打断、背景人声等。
    • 高质量 TTS 训练数据合成
    • 开源了 StepEval-Audio-360 测试集：137 条多轮对话，中文。数据涵盖9个维度，语音指令遵循、语音理解、逻辑推理、角色扮演、创作能力、唱歌、语言能力、语音情感控制、游戏，GPT-4o 进行事实性、相关性、对话评分；人类评估自然度、任务完成度。

1. 系统架构

  • Audio Tokenizer
    • 两种 token 的 next token 预测，训练 loss 详见论文图4，相较于单码本，收敛时语言学、语义 token loss 分别从 3.3、1.95 左右下降到 2.7、1.8 左右。
  • LLM 生成文本。初始模型为 Step-1 130B。
    • 出发点：高质量纯语音对话数据的稀缺性，且场景有限。
  • Speech Decoder：Step-Audio-TTS-3B LM + flow matching + vocoder。

2. 训练

2.1. Continue Pretrain

2.1.1. 数据集

  • 音频约 730wh，TTS 合成音频约 70wh，ASR 约 65wh，音频文本交错数据约 200wh。
  • 文本：800B token
  • 图像：800B token，图像文本成对/交替数据。

2.1.2. 训练

  • Stage1: 添加 5120 个音频 token，LLM 主干的学习率始终较少（2e-5）以保留其能力，embedding、LM head 学习率为主干的5倍。图像 encoder 冻结。音频:文本:图像=2:1:1，仅采用纯音频延续任务，1.2T tokens。
  • Stage2: 音频:文本:图像=2:1:1，音频延续数据:音频-文本交错数据=1:1，800B tokens。
  • Stage3: 音频:文本:图像=4:3:3，音频延续数据:音频-文本交错数据:ASR:TTS=1:1:1:1。LLM 整体 cosine schedule，2e-5 降到 5e-6。

2.1.3. 基础设施

数千个 H800 GPU。优化：

  • 定制化的 GPU kernel；
  • 通信重叠；
  • 引入 StarWeaver（RPC-based 分布式数据处理），将 CPU 数据预处理转移到远程进程中，同时实现了更好的负载均衡；
  • 针对多模态 encoder、LLM 异构，定制模型并行策略，减少流水线 bubble。
    • Zhang Z, Zhong Y, Ming R, et al. Disttrain: Addressing model and data heterogeneity with disaggregated training for multimodal large language models[J]. arXiv preprint arXiv:2408.04275, 2024.

2.2. Post-Training

2.2.1. TTS

2.2.1.1. 数据集

  • Step-2 LLM 生成文本，使符合语种、方言、风格。
  • 采用预训练的 Step-Audio checkpoint，audio-token cooldown 机制，生成特定说话人、语种、方言的音频。
    • 语种、方言：收集 native speaker 的语音、文本作为提示，快速扩展 native speaker 数据量。
    • 目标说话人：模型泛化语种、方言能力有限，无法达到母语者水平，选取与目标说话人音色、韵律相似的 native speaker ，与目标说话人的提示音频结合，重新生成音频 token。
  • SFT，得到 Audio-Edit 模型，对副语言进行精确控制。
    • 收集同一说话人、同一文本，正常、有情感的语音对，采用 (text, neutral audio token, emotion and style audio token) 进行 SFT。
  • 构建 歌词 - 干声/RAP 数据集
    • 收集 1wh+ 带歌词时间戳的曲目；
    • 采用 Demucs 提取干声，采用 VAD 去除静音段；
    • 分段；
    • 识别说唱片段：语速快、流派分类模型；
    • 数据清洗
      • 保留低噪声的单说话人片段；
      • ASR，对比歌词 CER，丢弃对不齐的片段。
  • 数据质量客观评估：ASR、VAD、说话人分离准确度、情感识别一致性、DNS（噪声抑制）

2.2.1.2. 训练

  • SFT 数据格式：两回合对话，其中第一回合 文本-音频 作为提示。
  • 指令 tag：描述性 tag 用于指定语种、方言、音色、风格，比较性 tag 用于指定情感、语速的等级。前者数据由 Step-Audio 生成，后者由 Audio Edit 生成。

2.2.2. AQTA

  • SFT
    • 数据：TQTA、AQTA、TAQTA、AQAA、VAQTA、ASR
    • 修改输出使更符合对话风格。多轮对话历史中的语音改用文本。
  • PPO
    • reward model
      • TQTA 人类偏好数据，移除语音对话中较少出现的类别，如代码，主要保留日常对话、角色扮演、安全、指令跟随等类别。
      • AQTA：从用户收集。针对指令跟随、对话自然度、安全性进行人工打分，用 LLM 对客观问题的响应正确性打分。
      • SFT 初始化，TQTA 预训练，AQTA 微调，Bradley-Terry loss，\(P(i>j)=\frac{s_i}{s_i+s_j}\)。
      • 将音频清晰、回复包含 “I didn’t hear clearly” 等存在 deaf hacking 现象的样本作为负样本。

3. 评价

ASR任务，大多数测试集上，相较于采用离散音频 token 的模型，采用连续表示性能更好。

TTS 任务，相较于单码本，采用双码本，内容一致性有提升，说话人相似度下降。扩大 TTS LM 规模，内容一致性可进一步提高。
