2. 语音对话系统的能力

  • 理解音频中的语言、副语言、声学信息，并相应地生成响应。如 E-chat [227] 中一个经典的示例，如果用户以愉快的语气问“我的手机无法开机，我该怎么办？”，系统可能会回答“看起来你对购买新手机很兴奋。你对什么类型的手机感兴趣？”相反，如果用户以悲伤的语气问同样的问题，系统可能会回答“很遗憾你的手机无法使用。如果你已经熟悉了维修政策，让我们继续下一步。”。支持多语种，进一步的，自动以用户使用的语言进行响应。
  • 生成语音的可控性，如音色、风格、口音、情感、音高、语速等，或自动根据用户调整回复语音的风格。（音色克隆：提供该能力有安全性问题）
  • 音频、音乐理解与生成。
  • 长上下文多轮对话。
  • 流式生成，低延迟。
  • 全双工对话，支持 normal turn exchanges、打断、backchannel（不打断的反馈，如 OK、I see）。具体的，用户说完后立即响应、正确处理用户语音中间的停顿、检测用户的打断、用户说话时的 backchannel、必要时打断用户（是个真实需求吗？）。
  • 扩展更多模态与能力，如图像、视频等。
  • 安全性。

3. 系统结构

  • 模态 adapter：如 Window-Level Q-Former [121]。
  • 交错生成文本、语音，如 Spirit-LM 通过可视化表明采用这种数据格式训练，文本、语音特征的相似度更高。
  • 并行生成文本、语音，如 PSLM、Llama-Omni、Mini-Omni。

4. 语音表示

4.1. Semantic vs Acoustic

  • 语义
    • 自监督：如 HuBERT、Wav2Vec、XLS-R、WavLM、SPIRAL 等。
    • 有监督：如 Whisper、supervised semantic speech(\(S^3\)) tokenizer 等。
  • 声学：如 Encodec。
  • 语义 + 声学：如 SpeechTokenizer（RVQ-1 蒸馏 HuBERT）、mimi（split RVQ）。
  • 音频生成
    • 语义 token
      • 如 HuBERT + HuBERT unit-based HiFi-GAN，\(S^3\) tokenizer + flow matching + HiFi-GAN。
      • 存在的问题：语音、声音事件、音乐统一建模。
    • 声学 token: vocoder
    • LLM 隐藏表示
      • 如 Freeze-Omni（LLM 隐藏表示 + 输出文本 token -> NAR decoder -> AR decoder -> vocoder）、IntrinsicVoice (Group-Former)。
  • 副语言信息的理解：语音表示中蕴含或提供额外的表示，如 speech embedding、副语言属性等。
  • 展望：将语音表示对齐到 LLM 的文本表示空间，减轻 LLM 的生成负担。

4.2. 连续 vs 离散

  • CV 领域整合离散、连续表示的工作
    • Zhou C, Yu L, Babu A, et al. Transfusion: Predict the next token and diffuse images with one multi-modal model[J]. arXiv preprint arXiv:2408.11039, 2024.
    • Show-o: One single transformer to unify multimodal understanding and generation.

4.3. 量化器 单层 vs 多层

  • RVQ 多层表示：delay 并行生成、moshi depth transformer 并行生成。
  • 单层：如 WavTokenizer [90], Single-Codec [119], BigCodec [224]。

4.4. 是否包含文本指导

  • 在语音输入、语音输出中间插入文本任务，类似 chain-of-thought。

5. 多阶段训练

  • 对齐语音-文本模态以保证一致的理解与生成。
  • 偏好优化（RL）
    • DPO：如 Qwen-Audio 2、Align-SLM（semantic metrics）。
    • REINFORCE：如 Seed-TTS。

6. 流式

  • 模型结构优化：因果卷积、chunk-wise causal attention。
  • 长序列生成优化：如 speech tokenizer 帧率压缩、IntrinsicVoice GroupFormer。
  • 流式音频生成 如 Encodec、LLama-Omni。另外，流式 ASR 如 U2++ Conformer、流式 TTS 如 XTTS-v2。

7. 全双工

7.1. 级联系统

  • Duplex Conversation [130]：3个核心模块
    • 用户状态检测：多模态模型，输入音频和文本，结合韵律、音高、停顿等特征，检测用户意图为 switch turn、继续说话或犹豫。
    • 响应信号选取：采用 multi-label 分类，在适当的时间插入适合对话情境的 backchannel。
    • 打断检测：多模态模型，输入音频和文本，识别用户中断，并避免将背景噪声、非预期的声音信号误判为中断。
  • Outbound Agent System [98]：采用有限状态机。
    • 采用基于 BERT 的流式文本分类器，每 300ms 对 ASR 结果进行语义分析，以识别中断（并避免 backchannel 被识别为中断）。
    • 处理用户提及命名实体时的停顿、不连续表达，避免被检测为对话结束。
  • Full-duplex LLM [211]：采用神经有限状态机、流式 ASR、流式 TTS。
  • CleanS2S [159]：VAD + ASR + LLM + TTS。

7.2. 端到端系统

如 dGSLM、Moshi、SyncLLM、OmniFlatten。

9. 评价

其中，SUPERB、AudioBench、AIR-Bench Foundation、SD-Eval、MMAU 主要为语音、音频、音乐理解相关任务，VoiceBench、AudioBench、AIR-Bench chat 还包含一般知识、指令跟随相关任务。VoiceBench 还评价在不同说话风格、声学环境、口语错误表达下的鲁棒性。SpokenWOZ 评价任务导向的口语对话中能否正确完成用户请求。SuperCLUE 非开源。

维度 评价方式与指标
文本智能 测试集如 知识问答（TriviaQA）、专业考试（MMLU、C-Eval）、数学推理（GSM8K）、指令跟随（Flan、Self-instruct）、多轮对话（CoQA、OpenAssistant）。
封闭式问题，如多项选择、给定答案关键词，可采用准确度相关的评价指标。
开放式问题，可采用机器翻译中 BLEU, METEOR, ROUGE 等文本相似度指标；或者 BertScore 等语义相似度指标；或者采用 GPT4o 等 LLM 评价生成文本是否符合人类偏好。
语音智能 能否基于不同的声学信息生成适当的响应。
音频智能 音频理解与生成：如语音情感识别准确度、自动音频字幕、声音事件检测、声学场景分类、音乐相关任务等。
语音生成质量 发音准确度（WER/CER）、清晰度和自然度（MOS）。
多语种能力 语音翻译任务、能否自动以用户使用的语言进行响应。
context learning 记忆能力、用户修改关键信息后的正确响应。
延迟 流式：first token delay（用户说完话到收到第一帧回复语音的延迟）。
turn-based：RTF。
交互能力 用户打断，适当的 backchannel、打断用户。
