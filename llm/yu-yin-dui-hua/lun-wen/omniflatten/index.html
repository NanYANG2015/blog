OmniFlatten

LLM 语音对话 论文
发布日期:   2024-11-06
更新日期:   2025-02-07
  • Zhang Q, Cheng L, Deng C, et al. OmniFlatten: An End-to-end GPT Model for Seamless Voice Conversation[J]. arXiv preprint arXiv:2410.17799, 2024.
  • https://omniflatten.github.io/
  • 作者：来自阿里通义实验室
  • 创新点：不进行计算密集型的预训练，而是通过 10wh ASR、TTS 任务进行模态对齐。相较于 SyncLLM，不对音频离散 token 去重，避免重建时引入的误差导致退化。

1. 系统结构

  • 离散语音 tokenizer: 同 CosyVoice，单码本 4096，25Hz。
  • Qwen2-0.5B。
  • 采用固定大小的文本 token chunk size = 2、音频 token chunk size = 10。文本结束后填充 \(\text {silent_text_token}\)，音频流填充 \(\text {silent_speech_token}\)。

2. 训练

多阶段后训练 数据 数据集 训练策略
ASR + TTS \([ASR][SOS] \mathrm{S} _ \text {seq}[EOS][SOT] \mathrm{T} _\mathrm{seq}[EOT] \\ [TTS][SOT] \mathrm{T} _\mathrm{seq}[EOT][SOS] \mathrm{S} _ \text {seq}[EOS]\) 10wh 中英，Aishell-3 + LibriTTS + TED-LIUM + VoxPopuli + Librispeech + MLS + 自有数据集 最大序列长度 1024。
flatten 4 流 用户+系统 文本+音频 36w turn-based 多轮对话，2kh，文本对话数据集 TTS。 最大序列长度 8192。
mask 用户通道 loss 训练更稳定，可能由于用户通道进行了加噪。
flatten 3 流 上述去除用户文本
flatten 2 流 仅音频

* batch size: 1亿 token。

  • 训练框架：NanoGPT
  • 语音对话数据集构建：高质量文本对话数据集 TTS
    • Alpaca、Moss、BelleCN、ultraChat
      • 过滤：含大量非文本元素（如代码、数学表达式）、中英文超过 200 个单词、包含不常见的符号等。
    • CosyVoice TTS
      • 用户说话人 embedding 从 Librispeech、3DSpeaker 中采样。用户通道加噪，MUSAN，SNR [15, 25]dB。
    • 用户语音结束后助手立即回复，助手语音结束后、用户开始说话前采样停顿时长。

3. 评价

* QWen-max

  • Assistant Turn-taking Acc@k: 用户有语义的语音 token 结束后第 k 个 token 助手正确预测为非 silent token。
  • User Turn-taking Acc@k: 助手说话时用户输入有语义的语音 token，第 k 个 token 后助手正确预测为 silent token。
  • taking turn 平均响应时间：系统 160ms，用户 805ms。训练集未覆盖用户打断助手语音的数据。
