AudioLDM 2

LLM 音频生成 论文
发布日期:   2024-07-03
更新日期:   2024-07-03
  • 作者：Surrey 大学、字节跳动等。
  • 特点
    • 语音、音乐、音效 生成统一框架。有条件生成。
    • 类似 MusicLM、AudioLM，分别采用语义 token、声学 token，本文中分别对应 AudioMAE、VAE 表示。
    • 生成流程
      • 采用多个 encoder，将多种条件拼接、输入 GPT-2；
      • GPT-2 生成语义 token（AudioMAE）；
      • 采用 LDM，以语义 token 为条件，生成声学token（VAE 表示）；
      • 采用 VAE decoder 重建 mel 谱；
      • 采用 HiFiGAN vocoder 生成音频采样点。
    • 性能：text-to-audio、text-to-music SOTA，text-to-speech 显著优于 FastSpeech2。

1. 系统结构

H0=G∘M:C↦Y^↦x\mathcal{H}_0=\mathcal{G} \circ \mathcal{M}: C \mapsto \hat{Y} \mapsto x H0​=G∘M:C↦Y^↦x

Y=A(x)Y=\mathcal{A}(x) Y=A(x)

其中，CCC 为 condition，可以是文本、音素、音频、图像等。Y^\hat{Y}Y^ 为音频中间表示，xxx 为采样点。A\mathcal{A}A 为自监督表示学习预训练模型，如 AudioMAE。M:C↦Y^\mathcal{M}: C \mapsto \hat{Y}M:C↦Y^ 采用 GPT-2。Y^↦x\hat{Y} \mapsto xY^↦x 采用 latent diffusion 模型（VAE 表示） + VAE decoder（mel 谱） + HiFiGAN vocoder（采样点）。

1.1. AudioMAE

  • 相较于其它自监督预训练模型的优势
    • 相较于专注特定领域的模型，如 MERT（音乐）、HuBERT（语音），AudioMAE 在 AudioSet 多种音频上进行了预训练，在通用音频分类任务上具有 SOTA 性能。
    • 相较于采用对比 loss、next token 预测等判别性损失，如 wav2vec，AudioMAE 采用重建损失，可能更适合生成任务。
  • audio masked autoencoder (AudioMAE) encoder：将 log mel 谱，采用 k=s=P,c=Dk=s=P, c=Dk=s=P,c=D的卷积，分割成 P×PP \times PP×P 的 patch，encoder 输出 E∈RT′×F′×DE \in \mathbb{R}^{T^{\prime} \times F^{\prime} \times D}E∈RT′×F′×D，T′=T/PT^{\prime}=T / PT′=T/P，F′=F/PF^{\prime}=F / PF′=F/P。
  • 后处理：2D average-max pooling[52]（保留时频关系）k=s=λk=s=\lambdak=s=λ，reshape，得到 embedding 序列 Lλ×DL_\lambda \times DLλ​×D，Lλ=T′F′/λ2L_\lambda=T^{\prime} F^{\prime} / \lambda^2Lλ​=T′F′/λ2。

1.2. VAE

  • variational autoencoder (VAE) 用于 LDM。V:X↦z↦X^\mathcal{V}: X \mapsto z \mapsto \hat{X}V:X↦z↦X^，压缩 mel 谱、重建。
  • 优化目标：重建损失 + 判别损失 + zzz 和标准高斯的 KL 散度（约束 latent 空间的方差）。

1.3. AudioMAE vs VAE

  • 相较于 AudioMAE，VAE 更关注重建质量，压缩比更高。
  • AudioMAE 包含更多语义信息。

1.4. GPT-2

  • condition
    • 可以是文本、音素、音频、图像等。采用多个 encoder + 线性层，沿序列长度维度拼接多个 encoder 表示。
      • contrastive language and audio pretraining (CLAP)：音频字幕（不同于转写）采用 CLAP text encoder，否则采用 CLAP audio encoder，如 TTS（同 AudioLDM）。
      • FLAN-T5（enhanced text-to-text transfer transformer (T5)）：用于捕捉文本的时序信息。
      • 音素 encoder：见 NaturalSpeech [2]。采用 Espeak phonemizers 将文本转为音素序列。
    • 除音素 encoder 无可用预训练权重外，其它预训练的 encoder 冻结。
    • TTS 训练时，采用 CLAP audio encoder 计算 condition，推理时采用 CLAP text encoder，有利于 prompt-based 说话人控制。
  • 采用连续表示 AudioMAE 的 MSE loss 微调。
  • 由于序列长度变短，相较于 MusicLM、SoundStream，减轻了推理计算成本高、错误累积的问题。

1.5. LDM

  • latent diffusion model (LDM) 以 AudioMAE 序列为条件，采用 VAE 表示进行特征压缩。
  • 采用 Transformer-UNet (T-UNet) 架构。为了扩大模型容量，在卷积 block 后插入了 ntrans +1n_{\text {trans }}+1ntrans ​+1 个 transformer blocks，最后1个 transformer block 采用 cross attention 引入 condition 信息，其它采用 self attention。
  • 在 TTS 任务中，添加额外的 cross attention 层，引入 FLAN-T5 condition。

1.5.1. Classifier-free guidance

  • 训练时以一定概率（如0.1）丢弃 condition。生成时 $$\mathcal{G}^{\prime}\left(z_t, t, Y ; \phi\right)=w \mathcal{G}\left(z_t, t ; \phi\right)+(1-w) \mathcal{G}\left(z_t, t, Y ; \phi\right)$$
  • Denoising Diffusion Implicit Models (DDIM) [90] 采样时 classifier-free guidance scale 设为 3.5。

3. 训练

  • AudioLDM 2 T-UNet ntrans=2n_{\text {trans}}=2ntrans​=2，AudioLDM 2-Large ntrans=6n_{\text {trans}}=6ntrans​=6。
  • AudioMAE：P=16P=16P=16，10s 音频序列长度 512，768 维。最后 16 个 transformer 层输出求平均，作为最终的表示。
  • 训练时，按一定概率动态选择 AudioMAE 特征来自 ground truth（Pgt=0.25P_{\mathrm{gt}}=0.25Pgt​=0.25）还是 GPT-2（Ppred =0.75P_{\text {pred }}=0.75Ppred ​=0.75）。
  • GPT-2、LDM 分别训练，再联合微调。音频-文本配对数据训练 GPT-2。所有音频数据训练 LDM。
  • LDM 预训练：用 10s 片段训练，零填充到10.24s。随机选择 λ∈{1,2,4,8}\lambda \in\{1,2,4,8\}λ∈{1,2,4,8} 以增强鲁棒性。

  • 评价指标

      • Frechet Audio Distance (FAD)：VGGish 模型提取的目标、生成音频的分布距离。
      • KL 散度：采用 Patch-out Transformer（音频 tag 模型），评价目标、生成音频的相似度。

    • 主观
      • 总体印象（OVL）、音频-文本相关性（REL）、MOS（TTS）。
      • 众包标注，可靠性：
        • worker: approval rate >= 60%，approvals >= 50。
