AudioPaLM

LLM 音频生成 论文
发布日期:   2024-05-10
更新日期:   2024-07-11
  • 作者：Google，2023.06。
  • 功能：speech-to-speech translation、speech-to-text translation、ASR、MT、TTS，多语种。

1. 系统结构

  • 输入：在输入前加任务 tag 前缀，以及输入语种、输出语种（可选，若与输入语种不一致）。任务 tag 采用 normal text，而非 special tokens。输入语种对低资源语种有益。
  • audio encoder：USM-v2，相较于 USM-v1 2B，用辅助 ASR 损失训练、微调。
  • 文本 tokenizer：SentencePiece。
  • LLM：PaLM-2 8B。
  • audio 合成：SoundStorm 或 AudioLM。

1.1. audio encoder

  • 从 audio encoder 中提取 embedding，并通过 k-means 量化为离散 token。25Hz，词汇表大小 1024。
  • 与原始 AudioLM 的区别：在多语种数据上训练 w2v-BERT；k-means 聚类前未对 embedding 归一化，归一化降低性能。

1.2. LLM

  • 用 PaLM-2 8B 初始化，扩充 embedding 矩阵维度，audio embedding 全 0 初始化，输入、输出 embedding 矩阵共享，训练所有模型参数。

1.3. audio 合成

  • 自回归解码：AudioLM stage2+3，stage2 输入 LLM 生成的 token 及说话人语音 condition，自回归生成粗粒度 SoundStream token；stage3 进一步生成细粒度 SoundStream token。
  • 非自回归解码：SoundStorm，并行处理所有 token。相较于 AudioLM，音质相当，语音、声学环境的一致性更高，合成速度快2个数量级。
  • 上述两种方案，均先将音频 token 转换为 SoundStream token，然后采用卷积 decoder 转换为波形。
  • 将 3s 长的语音作为 condition，以保留说话人的声音；若语音小于3s，重复音频。
  • 在 Multilingual LibriSpeech 上训练。

2. 数据集

2.1. 训练集

  • CVSS：扩展了CoVoST2，合成了目标文本的语音。CVSS-C：采用1个标准说话人的声音；CVSS-T：采用源语音的声音。
  • VoxPopuli：转写翻译音频。
  • conversational dataset：众包朗读 西班牙语-英语机器翻译 中的西班牙语部分。
    • Leveraging weakly supervised data to improve end-to-end speech-to-text translation
  • YouTube ASR：采用 USM-2B ASR 模型转写。
  • WMT/TED TTS：采用 WMT、TED 文本翻译数据集，TTS 引擎合成成对音频。
  • PaLM MT TTS：采用 PaLM-2 将 YouTube、Common Voice、Babel 数据集翻译为英文，再采用 AudioPaLM 8B S2ST 模型合成语音。

2.2. 测试集

  • AST：CoVoST2、FLEURS，SacreBLEU corpusBLEU，不进行任何 TN。
  • ASR：VoxPopuli、CoVoST2，JiWER，忽略大小写、标点，日文、中文评价 CER。
  • S2ST：CVSS，将模型输出的音频输入 ASR 模型，计算识别文本与 ground truth 文本的 BLEU 分。

3. 评价

  • 模型
    • AST 模型：ASR + AST
    • S2ST 模型：ASR + AST + TTS + S2ST
  • 训练参数：Adafactor 优化器、常数学习率 5e-5。
结果 结论
文本质量
级联方法：采用 AudioPaLM-2 ASR 模型 + 另一个仅采用 CoVoST2 T2T 翻译微调了的 AudioPaLM-2 模型。		 AST、S2ST 优于基线；ASR competitive。
语音质量
客观评价：
 音质：no-reference MOS estimator [Reddy et al., 2021]；
 语音相似度：采用说话人确认模型，计算 embedding 的余弦相似度，源语音采用 SoundStream 编码/解码；
 声学一致性：采用 [Borsos et al., 2023]，判断音频片段是否属于同一录音的模型，计算 embedding 的余弦相似度。		 相较于 Translatotron 2、CVSS-T ground truth 合成音频，MOS 得分显著地更高。
对于高资源语种、低资源语种，上述指标无明显差异。
zero-shot AudioPaLM-2 的文本翻译能力迁移到了音频域。

3.1. 消融实验

结果 结论
多任务训练 ASR 任务有助于提升 AST 性能。
Training from scratch vs. finetuning 微调的性能显著较好。
audio tokenizer USM-v2 > USM-v1 > W2V-BERT
组合任务 对于复杂的组合任务（如AST、S2ST），类似 chain of thought prompting，使模型也输出中间结果，如：将 [S2ST English French] 任务扩展为 [ASR AST S2ST English French]。
组合任务可以提高目标任务的性能，模型可以关注输入 + 前面的解码结果。
ASR 性能降低：可能和 checkpoint 选取等因素有关。
S2ST 添加 S2ST 任务，ASR、AST 任务性能略有下降。模型容量有限，必须学习输出音频。
扩充训练集 性能提升。由于 checkpoint 选取等因素，ASR 性能略有下降。
音频解码（从音频 token 重建波形） SoundStorm > AudioLM，相较于 AudioLM，SoundStorm 产生更可懂的语音。
PaLM vs PaLM 2（在翻译数据上进行了训练） AST 性能变好；ASR 性能变化不一致。
模型大小 性能随模型尺寸的增大而提高。
