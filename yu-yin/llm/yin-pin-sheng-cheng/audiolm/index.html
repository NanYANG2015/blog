AudioLM

语音 LLM 音频生成
发布日期:   2024-05-22
更新日期:   2024-05-22
  • 作者：Google, 2022.09。
  • 功能：输入 prompt 音频（3s），生成延续，在语法、语义上连贯，同时保持说话人的音色、韵律、录音环境。还能扩展到钢琴音乐延续。
  • 创新点
    • 音频表示同时采用语义 token、声学 token，兼顾了长时结构的一致性（如语音中的语法、语义，音乐中的旋律、和弦、节奏）、高合成质量。
    • 采用3阶段 decoder-only Transformer LM，分别建模语义、粗粒度声学表示、细粒度声学表示。每个阶段的 token 序列长度更小。
  • 优点
    • 不需要带标注的训练音频。

1. 系统结构

  • 音频表示：离散 token
    语义 token：采用自监督表示学习 w2v-BERT，表征局部内容、长时结构；
    声学 token：采用 neural audio codec SoundStream，实现高质量的合成。
  • w2v-BERT XL 0.6B
    • 基于 Conformer，MLM loss + 对比 loss。40ms 帧率。
    • 中间层表示（MLM 第7层）k-means 聚类。聚类前进行归一化（各维 0均值、单位方差）可以显著地提高音素区分性。
  • SoundStream
    • residual vector quantizer (RVQ) 由 Q 个词表为 N 的量化器组成。16kHz、20ms 帧率，比特率为 50∗Q∗log⁡2Nbps50 * Q * \log_{2}{N} \mathrm{bps}50∗Q∗log2​Nbps。
    • 重建损失 + 对抗损失，端到端训练。
  • decoder-only Transformer 自回归 LM。

1.1. 3 阶段建模

3 个 decoder-only Transformer。

  1. 语义建模：自回归预测语义 token p(zt∣z<t)p\left(z_t \mid z_{<t}\right)p(zt​∣z<t​)。
  2. 粗声学建模：以语义 token 为条件，只建模粗声学 token（前 Q′Q^{\prime}Q′ 个量化器），恢复说话人、录音环境等声学特性。token 序列为 (z1,z2,…,zTS,y11,y12,…,y1Q′,y21,y22,…,,y2Q′,…,yTAQ′)\left(z_1, z_2, \ldots, z_{T_S}, y_1^1, y_1^2, \ldots, y_1^{Q^{\prime}}, y_2^1, y_2^2, \ldots,, y_2^{Q^{\prime}}, \ldots, y_{T_A}^{Q^{\prime}}\right)(z1​,z2​,…,zTS​​,y11​,y12​,…,y1Q′​,y21​,y22​,…,,y2Q′​,…,yTA​Q′​)，建模 p(ytq∣z,y<t≤Q′,yt<q)p\left(y_t^q \mid z, y_{<t}^{\leq Q^{\prime}}, y_t^{<q}\right)p(ytq​∣z,y<t≤Q′​,yt<q​) q≤Q′q \leq Q^{\prime}q≤Q′，其中，yyy 序列为 y+oy+oy+o，oi=(i−1 mod Q′)⋅No_i=(i-1 \bmod Q^{\prime}) \cdot Noi​=(i−1modQ′)⋅N。训练时 y11y_1^1y11​ 为第一个预测的 token。
  3. 细粒度声学建模：p(ytq∣y≤Q′,y<t>Q′,yt<q)p\left(y_t^q \mid y^{\leq Q^{\prime}}, y_{<t}^{>Q^{\prime}}, y_t^{<q}\right)p(ytq​∣y≤Q′,y<t>Q′​,yt<q​) q>Q′q > Q^{\prime}q>Q′，去除第二阶段有损压缩导致的 artifacts。
    由于细粒度声学 token 由粗粒度 token 局部确定，在 3s 的非重叠块上执行。因此，第3阶段不受目标音频序列长度影响，且可以采用更大的 Q 来实现更高的质量。
  • 条件独立假设：语义 token 与 声学 token 条件独立；细粒度声学 token 与语义 token 条件独立。
  • 优点：相较于语义 token、声学 token 交替，合并第2、第3阶段，该方案每个阶段的 token 序列长度减小了，计算更高效。

1.2. 推理

  • 无条件生成。无条件采样语义 token。可以生成具有不同说话人、韵律、声学环境特征的，语法、语义一致的，多样化的语音。
  • 声学生成。给定语义 token，合成语音的文本一致，具有不同的声学特征。
  • 延续。输入简短的 prompt 音频。

2. 实验

  • 训练集
    AudioLM 所有组件（w2v-BERT、k-means、SoundStream、decoder-only Transformer）的训练集都为 Libri-Light unlab-60k（英语）。
  • SoundStream
    Q=12Q=12Q=12，N=1024N=1024N=1024，Q′=4Q^{\prime}=4Q′=4。4个卷积 block stride (2,4,5,8)，音频采样率 16kHz，帧率 20ms。
  • decoder-only Transformer
    • 3个阶段的模型结构一致，各 0.3B，参数详见论文。
    • 3个阶段，音频随机裁剪到 30s、10s、3s。删除语义 token 的连续重复。
  • 训练
    训练音频 encoder、decoder -> 冻结，训练 LM。解耦、简化训练流程。
  • 推理
    3个阶段均采用采样，温度系数分别为0.6、0.8、0.6，权衡生成语音的多样性、语义一致性。
  • 对比语义 token、声学 token
    • 音素区分性
      • 方式：将 w2v-BERT k-means 聚类质心表示、SoundStream encoder 输出，转换为 RVQ embedding，计算 ABX 错误率。
      • 评价指标：ABX 错误率。考虑仅中心 phone 不同的 triphone，ABX 错误率为 X 更接近 B（中心 phone 不同）而不是 A（中心 phone 相同）的比例。
        • within-speaker: A、B、X 来自同一说话人。
        • across-speaker: A、B 来自同一说话人，X 来自另一说话人。
    • 重建质量
      • 方式：训练 SoundStream decoder，输入 token 重建音频。
      • 评价指标：ViSQOL score，参考音频、重建音频之间的感知相似度。
    • 训练 decoder-only Transformer，采用输出 token 合成音频。
      • 采用语义 token，音频合成质量很差。
      • 采用声学 token，保留了说话人特征、录音环境，但内容往往类似于 babbling。

3. 安全性

训练了分类器区分 AudioLM 生成的语音、压缩后的真实录音，准确率 98.6%。
