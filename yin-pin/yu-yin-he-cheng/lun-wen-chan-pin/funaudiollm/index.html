FunAudioLLM

音频 语音合成 论文&产品
发布日期:   2024-07-22
更新日期:   2024-10-29

    Du Z, Chen Q, Zhang S, et al. CosyVoice: A Scalable Multilingual Zero-shot Text-to-speech Synthesizer based on Supervised Semantic Tokens[J]. arXiv preprint arXiv:2407.05407, 2024.

  • 作者：阿里 通义语音组

  • 代码：https://github.com/FunAudioLLM

  • Demo：https://fun-audio-llm.github.io

1. SenseVoice

  • 能力：多语种语音识别、语种识别、情感识别、音频事件分类/检测、ITN 带标点。
  • label 制作
    • AED https://github.com/qiuqiangkong/audioset_tagging_cnn/tree/master
    • 语音情感识别 https://modelscope.cn/models/iic/emotion2vec_plus_large

1.1. SenseVoice-Small

  • 5个语种（中、英、粤语、日、韩）。支持音频事件分类（支持音乐、掌声、笑声、咳嗽、打喷嚏、喘息、哭），1句话内至多1个事件。训练集 30wh+。
  • 模型结构
    • 80 维 FBANK。堆叠连续帧，6 倍下采样。语音特征前拼接可选的 4 个任务 embedding：⟨LID⟩、⟨SER⟩、⟨AEC⟩、⟨ITN⟩ / ⟨NoITN⟩。
    • encoder: memory-equipped self-attention network (SAN-M)。
  • 训练
    • 训练时，以 0.8 的概率随机将 ⟨LID⟩ 替换为真实语种 token。使得推理时，既支持语种预测，也支持提示语种。

1.2. SenseVoice-Large

  • 50+个语种。支持音频事件检测（含起止时间，仅支持音乐、掌声、笑声）。训练集 40wh+。

    • SenseVoice 未采用专门的 AED 数据训练，而是采用 ASR 数据 + AED 伪标签训练。

2. CosyVoice

2.1. 功能

  • 多语种语音合成：中、英、日、粤语、韩。

  • CosyVoice-base-300M：支持 zero-shot in-context learning、跨语种音色克隆。提示语音 3s，克隆音色、情感、风格、韵律等。

  • CosyVoice-instruct-300M：支持生成富有表现力的声音，采用文本指令进行细粒度控制，如说话人，说话风格（包含情感、性别、语速、音高等），副语言特征（如插入笑声、呼吸、笑着说话、强调某些单词）等。

  • CosyVoice-sft-300M：在 7 个多语种说话人上进行了微调。

2.2. 系统结构

* CosyVoice 论文中，文本经过 BPE tokenizer 后、输入 LLM 前，会经过额外的 text encoder 模块，用于对齐语音、文本 token 的语义空间。在训练集文本上训练 word sentence-piece 模型，词典大小 4000。

* LLM 建模语义内容和韵律。说话人 embedding：x-vector，从预训练的voice-print 模型中提取。

* flow matching: ordinary differential equation based (ODE-based) diffusion model。相较于传统 diffusion 模型，加速了训练、推理。

* HiFTNet：支持流式生成。

*虚线模块是可选的，只用于特定的应用。

2.3. 离散 token

  • AudioPaLM 表明在语音翻译、ASR 任务上，采用辅助的 ASR loss 量化得到的离散 token，相较于原始 k-means 聚类，性能更好。

  • 初步实验表明，语音 tokenizer 的选择对系统性能，对数据数量、质量的要求至关重要。评价了 3 类：

    • 基于残差量化，如 SoundStream、Encodec、FunCodec。
    • 基于 multi-grouped 量化，如HifiCodec。
    • 语义 token，如 HuBERT。

    均无监督或自监督训练，因而与语义关联较少，导致合成不稳定，容易受数据噪声影响，需要大量干净的训练数据。

  • 本文提出 supervised semantic speech (\(S^3\)) tokenizer。采用预训练的 SenseVoice-Large，在 encoder 第6层后插入向量量化，单码本，大小 4096，1token / 20ms。提取的 token 和语音内容、副语言信息强相关。由于监督训练，增强了对数据噪声的鲁棒性、不依赖于干净的语音训练集。消除了文本转音素的需求。

  • 量化 \[\mu_l=\operatorname{VQ}\left(\mathbf{h}_l, C\right)=\arg \min _{\mathbf{c}_n \in C}\left\|\mathbf{h}_l-\mathbf{c}_n\right\|_2\] 其中，\(\mathbf{h}_l\) 为第 \(l\) 帧的隐层表示，\(C\) 为 codebook。
  • 训练时，codebook embeddings 采用指数移动平均更新。\[\mathbf{c}_{\mu_l}:=\alpha \mathbf{c}_{\mu_l}+(1-\alpha) \mathbf{h}_l\] 其中 \(\alpha\) 为预定义的衰减系数。
  • 之后加入额外的位置 embedding，增强时序信息。

2.4. 数据集

  • 训练集 17wh+

  • 数据集构建
    • 语音检测，信噪比（SNR）估计，说话人 diarization、分离。
    • ASR：SenseVoice-Large、Paraformer。采用强制对齐进一步过滤、增强标点的准确性。

2.5. 训练

  • text encoder + LLM

  • flow matching

    • 估计条件概率 \(P\left(S \mid X, v, S_{ref}\right)\)，其中 \(X\) 为语音 token（含提示语音 token + 生成语音 token），\(v\) 为说话人 embedding，\(S_{ref}\) 为参考语音的 mel 谱，用于进一步增强音色、声学环境一致性，\(S\) 为目标语音的 mel 谱。
    • 采用 classifier-free guidance、cosine scheduler、masked conditions。

  • zero-shot in-context learning

    * 其中，灰色方块表示在文本 token 和语音 token 间插入的 “turn of speech” token。

    • 若提示语音、输入文本同语种，拼接，作为 AR 模型输入，将提示语音 token 视为已经生成的。
    • 若提示语音、输入文本语种不同，丢弃源语种的提示文本、音频，避免源语种的韵律影响目标语种。
    • 提示语音对应的文本可以人工标注或 ASR。

  • CosyVoice-instruct

    采用指令数据微调 CosyVoice-base，不输入说话人 embedding。

CosyVoice-base 只输入文本。
文本 token、语音 token、训练集+模型规模 的影响
Exp-x-LibriTTS: CosyVoice 论文中在 LibriTTS 数据集上训练的小模型		 对比 Exp-1、Exp-2，相较于 HuBERT，采用 \(S^3\) tokenizer，合成语音的内容一致性显著提升；
对比 Exp-1、Exp-2、Exp-3，文本、语音 tokenizer 对内容一致性都很重要，但几乎不影响说话人相似度。

结合两者，ASR 性能显著提高；
添加 MLS 训练集文本的合成语音，显著提高了识别效果。可能表明对于 ASR，文本丰富度的重要性高于语音时长。
